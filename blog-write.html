블로그 글쓰기

작성 후 blog-draft.json을 다운로드하고, 프로젝트 폴더에서 python publish_blog.py --add로 게시합니다.

링크 넣는 방법 (3가지)

  • 버튼 — 「링크 삽입」클릭 → 글자·URL 입력 → 「본문에 넣기」
  • [[재충전소|https://url]] — 추천 (한글 앵커 쉬움)
  • [재충전소](https://url) — 마크다운 방식

기타 문법 ## 제목 · **굵게** · - 목록 · :::warn

미리보기

게시 순서 · 파일 위치

프로젝트 루트 (이 사이트 폴더):

D:\경락\커서\site setting\300site\4start
  1. 글 작성 — 이 페이지에서 작성 후 blog-draft.json 다운로드
    • 저장 위치: C:\Users\본인이름\Downloads\blog-draft.json (기본 다운로드 폴더)
    • 파일을 직접 옮길 필요 없음 — publish_blog.py가 Downloads에서 자동으로 찾습니다
  2. 게시 명령 실행 — PowerShell에서 프로젝트 폴더로 이동 후 실행: 
    cd "D:\경락\커서\site setting\300site\4start"
python publish_blog.py --add
    • draft가 다른 곳에 있으면: python publish_blog.py --add --draft "경로\blog-draft.json"
  3. 자동 생성·수정되는 파일 (위 폴더 안)
    • data\blog-posts.json — 글 목록 원본 (draft 병합)
    • blog\{글ID}.html — 글 상세 페이지 (예: blog\after-overtime.html)
    • blog.html — 블로그 목록
    • sitemap.xml — 검색엔진용 URL 목록
  4. Render 배포 — 같은 프로젝트 폴더에서 Git 푸시: 
    cd "D:\경락\커서\site setting\300site\4start"
git add .
git commit -m "블로그 추가"
git push
    • 푸시 후 Render가 자동 배포 → office-recharge-004.onrender.com

폴더 구조 요약

4start\
├── blog-write.html      ← 글 쓰는 페이지 (지금 여기)
├── publish_blog.py      ← 게시 스크립트
├── data\
│   └── blog-posts.json  ← 글 데이터
├── blog\
│   └── {글ID}.html      ← 생성된 글
├── blog.html            ← 블로그 목록
└── sitemap.xml
